## 当前已完成

- 前端已具备静态首页 `/` 与日报详情页 `/daily/YYYY-MM-DD`，日报内容继续以 `frontend/data/daily/*.ts` 为真源。
- 前端已具备 `/admin` 页面，用于读取内部监控数据；首页与日报详情页已接入最小 PV 上报。
- 后端已具备最小 FastAPI + PostgreSQL 监控骨架，当前接口包括 `/health`、`/api/admin/overview`、`/api/admin/pipeline-runs`、`/api/admin/errors`、`POST /api/traffic/pv`。
- PostgreSQL 初始化脚本与 seed 数据已具备，`pipeline_runs`、`run_errors`、`traffic_daily_stats` 三张表已定义，当前支持 seed 数据与新增真实记录并存。
- `backend/scripts/run_stage.py` 已具备最小运行记录包装层，用于在不接管原内容生产逻辑的前提下记录 stage 结果。

## 当前未完成

- 还没有部署实现、上线脚本、CI/CD 或正式环境编排。
- `/admin` 还没有登录鉴权、权限控制或内部访问隔离方案。
- 真实脚本运行状态与真实流量写库还没有形成正式生产闭环。
- 当前流量口径只到 PV，不包含 UV 去重、细粒度事件或更完整的数据治理。
- 生产环境变量、域名、数据库实例和跨域白名单仍需在部署前逐项确认。

## 当前部署目标

- 冻结一个可上线内测基线，而不是继续扩展功能。
- 前台公开范围只包括静态首页与日报详情页。
- `/admin` 仅作为内部监控入口使用，不作为公开产品能力。
- 后端只承载健康检查、只读 admin API 与最小 PV 写入，不扩展为业务主站。
- PostgreSQL 作为当前监控与最小流量记录的数据库，在部署前必须完成初始化并确认真实记录可写入。

## 当前明确不做

- 不新增前台或后台新功能。
- 不进入部署实现，不新增部署脚本、容器、基础设施配置或 CI/CD 文件。
- 不改现有前后端业务逻辑，不把站点改成 SSR，不改日报数据真源。
- 不把内容生产流程迁入后端，不把推荐与深读改成新的自动化脚本。
- 不在本里程碑内补做多用户、登录鉴权、支付、UV/Event 平台或更细粒度埋点。
